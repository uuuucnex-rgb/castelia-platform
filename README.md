# CASTELIA PLATFORM — прототип №2

Полноценная платформа виртуального дизайнера интерьера на материалах Castelia
(гибкий камень). Один сайт, **3 режима** под одной «кнопкой» на главном экране:

| Режим | Для кого | Что делает |
|-------|----------|------------|
| **✨ Дизайнер интерьера** | обычный пользователь («бабушка в 5 кликов») | фото → 5 простых вопросов карточками → ИИ подбирает материалы и пишет совет → визуализация на фото |
| **💬 Профессиональный ИИ** | тот, кто хочет «поговорить» | чат с ИИ-дизайнером: называешь стиль (например «лофт») — он сразу показывает подходящие материалы карточками, ты выбираешь → визуализация |
| **🖌️ Самостоятельно** | менеджеры / профи | загрузка фото, выбор до 3 материалов, рисование масок кистью, точечное нанесение + правка (это функционал прототипа №1 — MASK_STUDIO) |

> Прототип №1 (`../MASK_STUDIO`) оставлен как есть. Этот проект собран на его базе:
> дизайн, тема и режим масок сохранены, сверху добавлены два пользовательских режима и каталог из 15 коллекций.

## Запуск локально (этот Mac)

```bash
cd CASTELIA_PLATFORM
npm install
node server.js
# открой http://localhost:3003
```

Прототип №1 (порт 3002) и прототип №2 (порт 3003) могут работать одновременно.

### Остановить / перезапустить
```bash
# найти процесс
lsof -i :3003
# или просто Ctrl+C в терминале где запущен node
```

## Переменные окружения (`.env`)

```bash
# Картинки (маски / дизайнер / правка) — Google Gemini
GEMINI_API_KEY=AQ...                 # ключ уже прописан
GEMINI_IMAGE_MODEL=gemini-3-pro-image

# Текстовый ИИ (вопросы дизайнера + чат)
AI_TEXT_PROVIDER=gemini              # gemini | openai
GEMINI_TEXT_MODEL=gemini-2.5-flash

# Чтобы переключить текстового бота на OpenAI (как в ТЗ):
#   AI_TEXT_PROVIDER=openai
OPENAI_API_KEY=
OPENAI_CHAT_MODEL=gpt-4o

PORT=3003
```

**Переключение на OpenAI** (когда появится ключ): в `.env` поставьте
`AI_TEXT_PROVIDER=openai` и `OPENAI_API_KEY=sk-...`, перезапустите сервер.
Картинки всё равно генерит Gemini (gpt-image для интерьеров слабее).

Проверить конфиг: `curl http://localhost:3003/health`

## Деплой на домашний сервер (Docker)

На сервере должен быть установлен Docker + docker compose.

```bash
# скопировать папку CASTELIA_PLATFORM на сервер (scp / rsync / git), затем:
cd CASTELIA_PLATFORM
docker compose up -d --build
# приложение поднимется на http://<ip-сервера>:3003
```

`restart: unless-stopped` — контейнер сам поднимется после перезагрузки сервера.

Логи: `docker compose logs -f`
Остановить: `docker compose down`

> `.env` НЕ попадает в образ (см. `.dockerignore`) — он монтируется через
> `env_file`, поэтому ключи остаются только на сервере. Не коммитьте `.env` в git.

## Структура

```
/public
  index.html     — все экраны (главная + 3 режима + результат)
  style.css      — дизайн (тема Castelia из прототипа №1)
  catalog.js     — каталог 53 материалов (15 коллекций) + карта стилей + подбор
                   (используется и фронтом, и сервером)
  app.js         — ядро: роутер экранов, загрузка фото, лоадер, результат,
                   каталог-пикер + режим «Самостоятельно» (маски)
  designer.js    — режим «Дизайнер интерьера» (5 вопросов)
  chat.js        — режим «Профессиональный ИИ» (чат)
  logo.svg
server.js        — Express: картинки (Gemini) + текст/чат (Gemini/OpenAI)
package.json
Dockerfile, docker-compose.yml
.env             — ключи (не коммитить)
instruction.md   — полная спецификация
```

## API

| Эндпоинт | Назначение |
|----------|-----------|
| `POST /api/designer-recommend` | 5 ответов → подбор материалов + совет (текст ИИ) |
| `POST /api/chat` | чат-дизайнер → ответ + карточки материалов |
| `POST /api/generate-auto` | визуализация без масок (дизайнер / чат) |
| `POST /api/generate-masked` | визуализация по маскам (самостоятельно) |
| `POST /api/touchup` | локальная правка результата |
| `GET  /health` | статус и текущая конфигурация |

См. `instruction.md` для деталей промптов и контрактов.
