# CASTELIA PLATFORM — спецификация (прототип №2)

## Что это

Веб-платформа виртуального дизайнера интерьера на материалах Castelia (гибкий камень).
Собрана **на базе прототипа №1 (MASK_STUDIO)**: тот же дизайн, та же тема, тот же
функционал масок — сохранён и расширён. Прототип №1 остаётся отдельно как есть.

Главная идея: один сайт, на старте — выбор из **3 режимов** (одна «кнопка маски»
разделена на три сценария):

1. **«Дизайнер интерьера»** — для обычного пользователя. 5 простых вопросов
   карточками → ИИ-бот (на дизайне интерьеров) рекомендует материалы и как их
   сочетать → визуализация на фото. Цель: человек без опыта за ~5 кликов получает
   свой дизайн.
2. **«Профессиональный ИИ»** — чат с ИИ-дизайнером. Пользователь называет стиль
   («лофт») — бот сразу предлагает подходящие материалы Castelia карточками прямо
   в чате; выбранные материалы визуализируются на фото комнаты.
3. **«Самостоятельно»** — режим менеджера/профи: маски кистью (функционал
   прототипа №1) — точечное нанесение до 3 материалов + локальная правка.

---

## Стек

- **Frontend**: чистый HTML + Canvas + Vanilla JS (без сборщика), модули
  подключаются как отдельные `<script>` и общаются через `window.CST`.
- **Backend**: Node.js + Express (прокси к ИИ).
- **Картинки**: Google Gemini (`gemini-3-pro-image`) — генерация визуализаций.
- **Текст/чат**: Gemini (`gemini-2.5-flash`, thinking отключён) по умолчанию,
  переключается на OpenAI (`gpt-4o`) одной переменной `AI_TEXT_PROVIDER`.
- **Деплой**: локально (node) или Docker (домашний сервер).

## Каталог материалов

`public/catalog.js` — единый модуль (UMD): работает и в браузере, и в Node.
**53 материала, 15 коллекций** Castelia (источник картинок — репозиторий
`uuuucnex-rgb/casteliaCATALOG`):

Marble, Italian Travertine, Roman Pillar, Aerolite, Ando & Zen Cement,
Crude Wood, Rust Board, Foamed Aluminium, Golden Sunset, Natural, Dacite,
Line Stone Board, Terrazzo Rough, New Rock Cut Stone, Round Line Stone.

У каждого материала: `id, name, collection, family, tone, styles[], blurb, imageUrl`.

- `tone`: warm | cool | neutral | dark
- `styles`: loft, industrial, classic, minimalism, scandi, japandi, luxury,
  modern, mediterranean, eco

### Стили (для мастера) и карта «стиль → материалы»
`STYLES` — 8 стилей с превью-обложкой: Лофт, Классика, Минимализм, Скандинавский,
Японди, Лакшери, Средиземноморье, Эко.

`recommend(answers, count)` — детерминированный подбор (по стилю + желаемому тону +
разнообразию коллекций). Используется:
- на фронте как мгновенный фолбэк,
- на сервере как запасной вариант, если текстовый ИИ не ответил/вернул мусор.

## Режим 1 — «Дизайнер интерьера» (5 вопросов)

Поток: **фото → 5 вопросов → рекомендация → визуализация**.

5 вопросов (`QUESTIONS` в catalog.js), отвечаются нажатием на карточку (без ввода текста):
1. **Стиль** — карточки стилей с превью-картинкой материала.
2. **Помещение** — гостиная / кухня / спальня / ванная / прихожая / фасад.
3. **Атмосфера** — тёплая / светлая / статусная / брутальная (несёт желаемый `tone`).
4. **Палитра** — светлая / тёплый беж+дерево / серый бетон / тёмный контраст.
5. **Объём** — акцентная стена (1 материал) / вся комната (1) / сочетание 2–3 (3).

После 5 ответов → `POST /api/designer-recommend` → экран «Готово, вот ваша идея»:
название идеи + совет дизайнера + карточки материалов. Кнопка
«Показать на моём фото» → `POST /api/generate-auto`.

---

## Режим 2 — «Профессиональный ИИ» (чат)

- Приветствие → пользователь пишет (например «лофт в гостиной») или прикладывает фото 📎.
- `POST /api/chat` с историей (и фото при первом сообщении после загрузки).
- Бот отвечает тёплым текстом и **дописывает служебный тег** `[[MATERIALS: id, id]]`,
  по которому сервер отдаёт карточки материалов; на фронте они кликабельны.
- Выбранные материалы складываются в «трей»; при наличии фото — кнопка
  «Показать на фото» → `POST /api/generate-auto`.

---

## Режим 3 — «Самостоятельно» (маски)

Полный перенос редактора из прототипа №1:
- загрузка фото (ресайз до 1280 по большей стороне),
- до 3 слотов материалов (3 цветных слоя кисти),
- кисть/ластик/undo/очистка, активный слой подсвечивается,
- сбор одной **цветной маски** (red/green/blue по материалам) → `POST /api/generate-masked`,
- «Дизайнер сам разместит» (без масок) → `POST /api/generate-auto`,
- «Подправить» — локальная правка белой маской → `POST /api/touchup`.

## API-контракты

### `POST /api/designer-recommend`
```json
{ "answers": { "style":"loft", "room":{...}, "mood":{...,"tone":"dark"}, "palette":{...,"tone":"dark"}, "coverage":{"count":3} } }
→ { "materials":[{id,name,collection,imageUrl,blurb}], "title":"…", "advice":"…" }
```

### `POST /api/chat`
```json
{ "messages":[{"role":"user|assistant","content":"…"}], "image":{"mime","base64"}? }
→ { "reply":"…", "materials":[{id,name,collection,imageUrl,blurb}] }
```

### `POST /api/generate-auto` (дизайнер / чат)
```json
{ "baseImage":"data:…", "baseWidth", "baseHeight", "materials":[{materialUrl,materialName}], "hint":"Style: …" }
→ { "resultDataUrl":"data:…", "generationTime": 28450 }
```

### `POST /api/generate-masked` (самостоятельно)
```json
{ "baseImage", "baseWidth", "baseHeight", "combinedMaskImage", "materials":[{materialUrl,materialName,colorName}] }
→ { "resultDataUrl", "generationTime" }
```

### `POST /api/touchup`
```json
{ "baseImage", "baseWidth", "baseHeight", "materialUrl", "materialName", "maskImage" }
→ { "resultDataUrl", "generationTime" }
```

## Промпты ИИ (в server.js)

- **`designerSystemPrompt(count)`** — «ведущий дизайнер интерьера Castelia, 25 лет
  опыта…»: на вход ответы клиента + полный каталог, на выход строгий JSON
  `{materialIds, title, advice}`. Бот выбирает только из каталога; при сбое —
  детерминированный `recommend()`.
- **`CHAT_SYSTEM_PROMPT`** — «Кастелия, ИИ-дизайнер»: тёплый короткий диалог,
  при рекомендации дописывает `[[MATERIALS: …]]`. Знает весь каталог.
- **Картиночные промпты** (`buildMaskedPrompt`, `buildDesignerPrompt`,
  `buildTouchupPrompt`) — перенесены и усилены из прототипа №1: один фотореалистичный
  кадр, материал только на стены/фасады, сохранение объектов и геометрии.

> Важно для Gemini 2.5: текстовые вызовы идут с `thinkingConfig.thinkingBudget = 0`,
> иначе «мысли» съедают бюджет токенов и ответ приходит пустым.
